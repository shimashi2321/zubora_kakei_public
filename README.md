# ずぼら家計簿（Zubora Kakeibo）

都度の支出をいちいちメモせず、「月々の予算をざっくり決めて、給料日にまとめて処理する」だけのゆるい家計管理アプリです。
単一のHTMLファイルで動作し、データはお使いのブラウザの中だけに保存されます（外部送信なし・サーバー不要）。

## 公開しているファイル

| ファイル | 用途 |
|---|---|
| `zubora-kakeibo.html` | 通常版（読みやすいソースコード） |
| `zubora-kakeibo.obfuscated.html` | 難読化版（中身を見えにくくしたもの。機能は同じ） |

GitHub Pagesなどで配布する際は、見せたい方を `index.html` にリネームして使ってください。
両方公開しても問題ありませんが、コードを読まれたくない場合は難読化版だけを `index.html` にするのがおすすめです。

## できること

- **完全固定費**：家賃・スマホ・サブスクなど、毎月同じ金額のものを登録
- **定期的な貯金**：毎月「使わない」と決める金額を設定。給料日ごとに自動でたまっていく
- **半変動費**：食費・電気代など月によってばらつく費目。月末（締め）に「だいたいの金額」を入れていくと、これまでの最大月と最小月の平均が次月の予算として自動計算される
- **給料日の処理**：給料額を入れると「今月使っていいお金」が自動計算される（給料 − 固定費 − 半変動費予算 − 貯金）
- **都度の支出記録**：金額とメモだけのシンプル入力。残高からその場で引かれる
- **たまったお金**：定期貯金＋締め時に貯金へ回した分の累計。設定画面から直接修正も可能（修正履歴が残る）
- **締め処理の日付制限**：ONにすると、設定した給料日以外は締め処理ができないようにロックできる
- **データの書き出し／読み込み**：JSONファイルでバックアップ・機種変更に対応
- **締めデータの一括削除**：過去の締め履歴だけをまとめてリセットできる（設定・貯金合計は消えません）

## 使い方の流れ

1. **設定タブ**で給料日・固定費・貯金額・半変動費の項目名を登録する
2. 給料日になったら、**ホームタブ**の「💴 給料日の処理をする」を押す
   - （2回目以降）今月の半変動費の実績を入力 → 余ったお金のうちいくらを貯金に回すか決める
   - 新しい給料額を入力 → 「今月使っていいお金」が自動で計算される
3. 普段の買い物などは、ホームの「使った分を記録する」にその都度入力する
4. 残高（今月使っていいお金）は **月締めごとに0からリセット**され、前月の余りは持ち越されません。貯金に回した分だけが「たまったお金」として積み立てられます

## データの保存場所について

- すべてのデータはブラウザの `localStorage` に保存されます。サーバーには一切送信されません。
- ブラウザのキャッシュ削除や端末変更でデータは消えるため、**設定タブの「書き出す」でJSONファイルとして定期的にバックアップ**することをおすすめします。
- 機種変更時は、新しい端末で同じページを開き、「読み込む」から書き出したJSONファイルを選択してください。

## GitHub Pagesでの公開方法（例）

1. このリポジトリをGitHubに作成し、`index.html`（公開したいファイルをリネーム）をアップロード
2. リポジトリの `Settings` → `Pages` から、公開元ブランチ（例：`main`）とフォルダ（`/root` など）を選択
3. しばらくすると `https://(ユーザー名).github.io/(リポジトリ名)/` でアクセスできるようになります

## 注意事項

- 個人の家計管理を補助するための簡易ツールです。正確な会計・確定申告等の用途には対応していません。
- 半変動費の予算は「過去の最大月と最小月の平均」という簡易ロジックです。実際の支出を保証するものではありません。
